# Driver Drowsiness Detection System (MVP, KuickHack 2025)

## Описание

**Driver Drowsiness Detection System** — это MVP-прототип системы мониторинга усталости и отвлечённости водителей с использованием искусственного интеллекта. Проект создан специально для хакатона [KuickHack 2025](https://www.kuickhack.com) и оптимизирован для быстрой демонстрации на Windows без контейнеризации.

## Основные возможности

- Мониторинг состояния водителя в реальном времени (сонливость, отвлечённость, норма)
- AI-детекция: анализ видео с камеры, вычисление EAR (Eye Aspect Ratio), определение морганий, закрытия глаз, поворотов головы
- Веб-интерфейс для диспетчера: список всех водителей, их статус, подробная статистика по каждому
- Графики динамики: визуализация отвлечённости и сонливости по ходу сессии
- Автоматическая оценка водителя: цветовая индикация уровня риска, рекомендации
- Сохранение истории: все данные хранятся в базе данных (H2, встроенная)

## Как это работает

1. **Водитель** проходит идентификацию и запускает сессию мониторинга.
2. **Фронтенд** (MediaPipe/AI) анализирует видео с камеры, вычисляет признаки (EAR, моргания, повороты головы) и отправляет события на сервер.
3. **Бэкенд** сохраняет события в базе, агрегирует их по сессиям, рассчитывает статистику и формирует рекомендации.
4. **Диспетчер** через веб-панель видит список водителей, их статус, может открыть подробную аналитику по каждому: графики, оценки, сравнение с другими.

## Архитектура

- **Backend**: Java 17, Spring Boot, Spring Data JPA, H2 Database
- **Frontend**: Thymeleaf, Bootstrap 5, Chart.js
- **AI/Computer Vision**: MediaPipe (на фронте), OpenCV/DLib (опционально)

## Структура базы данных

- **drivers** — информация о водителях
- **driver_sessions** — сессии мониторинга (начало, конец, длительность)
- **driver_events** — события (сонливость, отвлечённость, метаданные, EAR, моргания и др.)

## Быстрый старт (Windows/без Docker)

1. **Клонируйте репозиторий:**
   ```bash
   git clone https://github.com/your-org/drowsiness-detection.git
   cd drowsiness-detection
   ```
2. **Запустите проект через Gradle:**
   ```bash
   ./gradlew bootRun
   ```
   Или откройте проект в вашей IDE (IntelliJ IDEA, VS Code) и запустите класс `DriverMonitoringApplication`.
3. **Откройте веб-интерфейс:**
   - Диспетчер: [http://localhost:8080/driver/login](http://localhost:8080/driver/login)
   - Диспетчер: [http://localhost:8080/dispatcher](http://localhost:8080/dispatcher)
   - H2 Console: [http://localhost:8080/h2-console](http://localhost:8080/h2-console)

## Как работает анализ и AI

- EAR (Eye Aspect Ratio) вычисляется на фронте, отправляется на сервер для анализа сонливости
- События типа DROWSY и DISTRACTED логируются только при отклонении от нормы
- Для каждого водителя и сессии считается процент времени в каждом состоянии, строятся графики
- Оценка формируется автоматически, цветом выделяется уровень риска

## Готовность к интеграции ИИ и сбору датасета

- Архитектура проекта полностью готова к замене rule-based алгоритма анализа на настоящую ИИ-модель после её обучения. Вся логика анализа вынесена в отдельные сервисы, которые легко заменить на вызов обученной модели (например, через REST API или локальный сервис).
- В проекте реализован специальный инструментарий для сбора маркированного датасета для обучения ИИ:
  - Все события с метками (LABEL) сохраняются в базе с подробным контекстом (EAR, blinkRate, состояние, история).
  - Для выгрузки датасета реализован API-эндпоинт:
    - **GET /api/label-dataset** — возвращает все маркированные события в формате JSON.
    - Датасет можно скачать через кнопку "Выгрузить датасет для обучения" на панели диспетчера или напрямую через API.
    - Пример запроса:
      ```
      GET http://localhost:8080/api/label-dataset
      ```
    - Ответ — массив JSON-объектов с признаками, метками и контекстом.
- Для обучения ИИ можно использовать простой Python-скрипт, который загружает JSON-файл с датасетом, преобразует данные в формат для обучения (например, Pandas DataFrame) и использует признаки и метки для тренировки модели. При необходимости можно реализовать скрипт, который будет напрямую подключаться к базе данных для получения контекста.

## Как работает rule-based алгоритм (имитация ИИ)

- На фронте и сервере реализованы простые правила:
  - Если EAR (Eye Aspect Ratio) ниже порога — считается, что водитель сонлив.
  - Если обнаружен поворот головы или длительное отсутствие взгляда на дорогу — считается, что водитель отвлечён.
  - События логируются только при отклонении от нормы (NORMAL не пишется).
- На сервере для каждого водителя и сессии считается процент времени в каждом состоянии, строятся графики и формируется итоговая оценка.
- Такой rule-based подход имитирует работу ИИ: он анализирует признаки и принимает решение на основе простых правил, которые в будущем заменит обученная модель.

## Примеры использования

- Мониторинг автопарка: диспетчер видит всех водителей, может быстро выявить тех, кто в зоне риска
- Аналитика по сменам: можно анализировать, в какие периоды смены чаще всего возникает усталость или отвлечённость
